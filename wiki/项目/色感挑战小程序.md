# 色感挑战小程序

> 微信小程序：色彩感知力测试 + 好友 PK。四种模式，段位系统，好友挑战裂变。

## 技术栈
- 微信小程序 + TypeScript
- 微信云开发（云函数 + 云数据库）
- Canvas 2D 分享卡

## 功能清单
- **色相排序**：将打乱色块按彩虹顺序排列
- **找不同色**：从网格中找出颜色差异的一块
- **颜色匹配**：从近似色中选出最接近目标色的选项
- **颜色记忆**：记忆颜色序列并复现
- **段位系统**：色感萌新 → 色彩学徒 → 色域行者 → 辨色大师 → 色感之神
- **排行榜**：今日排行 + 历史总榜
- **广告变现**：激励视频（复活/双倍）+ 插屏 + Banner
- **好友挑战**：生成挑战链接，好友扫码对战
- **新手引导**：每模式 3 步遮罩引导
- **分享卡**：Canvas 2D 生成分享图片

## 调试经验

### TypeScript 编译失败排查
- `btoa()` 不存在 → 手写 base64 编码替代
- `wx.arrayBufferToBase64` 不在类型定义中 → 有运行时 API 但 TS 编译不过
- 类型导入问题：`IAppOption` 必须 `export` 并在使用处 `import`
- 旧 `.js` 脚手架文件覆盖 TS 编译输出 → 删掉让工具重新编译

### CSS 变量失效
- WXSS `@import` 路径解析在不同版本开发者工具中不一致
- 解决：将所有 CSS 变量内联到 `app.wxss`，不使用 `@import`

### 全局防抖副作用
- 300ms `_tapGuard` 会拦截色块选择→槽位填入的连续快速操作
- 解决：只在需要防重复点击的模式（找不同/匹配/记忆）保留，色相排序去掉

### 字体 Emoji 渲染
- 微信小程序模拟器不完全支持 emoji 渲染
- 不影响真机，可直接忽略

## 拼豆模式 v6 重设计 (2026-06-02 晚)

在 v5 基础上进一步向真实拼豆体验靠拢。

### 设计决策（已确认）

| 决策 | v5 | v6 | 理由 |
|------|-----|-----|------|
| 布局 | 画布为主 | **纯画布模式**，参考图折叠为顶条 | 手机竖屏最佳利用 |
| 交互 | 逐点手工 | **三合一**：点击 + 画笔涂抹 + 区域填充 | 兼顾精准/效率/大面积 |
| 图案来源 | 12 个手绘 | **Emoji Canvas 像素化**，30-40 模板 | 系统字体渲染 → 降采样 → 色盘映射，形状准确 |
| 底板规格 | 16/24/32 | **29×29 / 40×40 / 48×48** | 参照真实拼豆 5mm 大豆 pegboard |
| 珠子渲染 | CSS border-radius | **Canvas 2D** 圆形 + 内孔 (2.3mm) + 3D 高光 + 阴影 | 真正拼豆质感 |
| 色盘 | 12 色 | **24 色** | 参照 Artkal/Perler 真实色盘 |
| 图案分类 | 无 | **Emoji / 萌宠 / 美食 / 自然 / 像素角色 / 几何 / 节日限定** | 8 大分类 |
| 解锁 | 无 | 29×29 默认 → 40×40 通关 → 48×48 段位 | 渐进难度 |

### Emoji 像素化流程

```
系统字体渲染 Emoji（290px 离屏 Canvas）
    ↓
读取像素 RGBA 数据
    ↓
按底板网格降采样（29/40/48 等分）
    ↓
每格取平均色 → 最近邻映射 24 色拼豆色盘
    ↓
过滤背景色（阈值 > 235 + alpha < 60）
    ↓
Canvas 2D 绘制完整珠子阵列
```

## 发布后修复 (2026-06-08)

### 兼容性
- **libVersion 降级**：3.16.1 → 2.30.4，覆盖 99%+ 用户
- **API 降级**：`wx.getWindowInfo()` → `wx.getSystemInfoSync()`（share-card.ts、sketch.ts）

### 添加好友修复
- **根因**：数据模型不一致，`addFriend` 写 `users.friends[]`，`getRanking friendsRank` 读 `friendships` 集合
- **修复**：重写 `addFriend` 云函数，双向写入 `friendships` 集合 + 兼容 `users.friends[]`
- **前端**：错误提示改为「请检查 addFriend 云函数是否已部署」

### 排行榜头像/昵称空白
- **根因**：字段名不一致（`submitScore` 写 `nick`/`avatar`，`profile` 写 `nickName`/`avatarUrl`）
- **修复**：
  - `getRanking` 日榜新增关联 `users` 表查询用户信息
  - `getRanking` 总榜/日榜统一归一化字段名
  - `submitScore`/`profile.ts` 双写两种字段名
  - `rank.wxml`/`friends.ts` 兜底读取两种字段名

### 统计图表不显示
- **根因**：`profile.ts` 设置 canvas 物理像素尺寸 + DPR 缩放后，chart 函数读取 `canvas.width`（物理像素）直接用作 CSS 像素坐标
- **修复**：`drawLineChart`/`drawRadarChart` 新增 `displayWidth`/`displayHeight` 参数

### 色相排序拖拽不跟手
- **3 个根因**：(1) touchStart 异步 `createSelectorQuery` 延迟 → `_cacheSortPositions()` 预缓存、(2) ghost 起始位置用元素坐标 → 手指居中、(3) ghost CSS `position:fixed` 缺 `top:0;left:0` → 补全
- **边缘卡顿**：slot 吸附新增滞后机制（`lastSnappedSlot` +15% 重叠加权）

### 色相排序答案错误
- **根因**：`generateHueColors` offset 随机到 `[0,360)` 导致色相值跨越 0°/360° 边界（如 200°→272°→344°→56°→128°）
- **修复**：offset 限制在 `[0, step)` 保证不绕圈
- **提示条优化**：6 段离散色块 → 完整 HSL `linear-gradient`，标注箭头方向

### 宣发素材
- `色感挑战-宣传页.html`：4 版海报（竖版主视觉 + 方形朋友圈 + 横版封面 + 信息图长图）
- 推广文案：3 版朋友圈 + 2 篇小红书图文（`docs/推广素材.md`）
